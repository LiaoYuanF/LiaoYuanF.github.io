前言

一点点用vector代替掉array，是我在开始学C++的过程中逐渐发生的，自己也没意识到，自己的代码中越来越少地出现以往十分依赖的array，而vector的使用频率逐渐高了起来。
今天写这篇blog，就是想讨论一下这两者之间的区别

首先，我先大略地总结一下这两者的相同点与不同点

数据结构 vector array
相同点 对同一种类型的数据进行储存 同
相同点 可以用下标操作进行 同
相同点 可以用迭代器进行操作 同
不同点 可以用size获取vector的长度 在定义时就已经确定了长度
不同点 长度不固定，可以随时增减 在定义后就无法更改
不同点 长度决定空间的占用大小，不浪费 会浪费额外的空间

最简单地概括，就是

vector是活的，array是死的。

也就是说，可以把vector理解为一个动态数组，弥补了数组动态分配空间的不便，更灵活
