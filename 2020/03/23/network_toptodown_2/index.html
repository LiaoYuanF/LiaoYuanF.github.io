应用层

应用层协议原理

应用程序体系结构

客户-服务器体系结构(C/S architecture)
依赖数据中心
经典的web应用程序

P2P体系结构
对数据中心低依赖
对等方之间直接通讯
因特网电话,迅雷

优势:可拓展性
挑战:ISP友好,安全性,激励

进程通信

通信的主体是进程,单个进程是但歌词运行在端系统中的程序
进程之间的通信过程:
1.进程两端为客户与服务器(下载端为客户端,上传端为服务器)
2.进程与网络的接口:socket
关于socket:开发者可以控制他在应用层的一切,但只能控制他在运输层的协议或是几个简单地运输层参数
3.进程寻址:通过IP与端口号进行寻址

应用程序的运输层选择

四个尺度评价运输层协议:
1.可靠性:能否提供可靠地数据传输
2.吞吐量:bandwidth-sensitive application 一般有着吞吐量要求,elastic application 一般没有硬性要求
3.定时:即对于延迟有着要求,典型的如同多方游戏
4.安全性:此处的安全性指的是该协议在运输过程中不会被第三方观察到

因特网提供的运输层服务

TCP:
面向连接服务:数据传输之前建立全双工连接,交换运输层的控制信息
可靠数据传输服务:无差错,有顺序,没有字节的冗余与丢失
拥塞控制机制:不一定直接有利于通信的进程,但有利于整个因特网网络

UDP:
仅提供最小服务,是无连接的

安全性:TCP与UDP都是明文,所以要通过额外的内容去实现,即 secure sockets layers (SSL), 这是一种TCP的加强,其实是在应用层上实现的

因特网不提供的运输层服务

很明显,TCP与UDP都没有对定时与吞吐量对作出处理

应用层协议

部分应用层协议是由RFC文档定义的,因此位于公共域
部分应用层协议是专用的

Web与HTTP

HTTP概况

Web是一个网络应用
HTTP是一个应用层协议,即为hypertext transfer protocol, 超文本传输协议

1.使用TCP作为运输层协议
2.使用C/S应用程序体系结构
3.是一个无状态协议:服务器不保存客户信息

非持续连接与持续连接

非持续连接:每一个seq,ack由一个单独的TCP连接发送
持续连接:所有的seq,ack由相同的TCP连接发送

1.采用非持续连接的HTTP
针对每一个请求的对象都要建立和维护一个TCP连接,服务器和客户都需要对此分配TCP的缓冲区和保持变量,会给服务器带来极大负担,同时每一个对象都会经受双倍RTT时延

2.采用持续连接的HTTP
HTTP的默认模式就是带流水线的持续连接,他可以避开非持续连接的那些缺点

HTTP报文格式
请求报文

请求行+首部行+实体主体
请求行:方法字段+URL字段+HTTP版本字段
方法字段-GET/POST/HEAD/PUT/EDLETE
首部行:
Host-指明对象所在主机
Connection
User agent-指明用户代理,即浏览器类型
Accept language-想要得到的对象的法语版本

响应报文

状态行+首部行+实体主体
状态行:协议版本字段+状态码+状态信息
状态码-200/OK,301/Moved Permanently,400/Bad Request,404/Not Found,505/HTTP Version Not Ssupported
首部行:
Connection
Date-服务器发送报文的时间
Server-类似于User agent
Last Modified-对象创建或者最后修改时间
Content Length:被发送对象的字节数
Content Type:对象的类型

用户与服务器交互:cookie

由四部分组成:
HTTP响应报文中的cookie首部行
HTTP请求报文中的cookie首部行
用户端保存一个cookie文件
web站点有一个后端数据库

用途:
web站点用来识别用户身份

Web缓存

即为代理服务器
浏览器的完整请求过程:
1.浏览器建立一个到Web缓存器的TCP连接,并向缓存器中的对象发送一个HTTP请求
2.web缓存器检查本地,是否有被请求的对象的副本,有的话直接回响应报文
3.如果没有副本,就与该对象的初始服务器进行TCP连接,缓存器对初始服务器发请求,之后收到响应
4.收到对象时,在本地存一个副本,然后响应客户

条件GET方法

作用:保证缓存器中的副本对象是最新的

文件传输协议:FTP

HTTP与FTP的区别:
FTP使用两个TCP连接,一个用来控制连接,一个用来数据连接
FTP服务器必须在整个会话期间保存用户的状态

电子邮件

组成部分

user agent
mail server
simple mail transfer protocol

SMTP协议

不使用中间邮件服务器

SMTP与HTTP的比较

这两个协议都用于两台主机之间的信息传输,都使用持续连接
区别:
HTTP是一个拉协议;SMTP是一个推协议
SMTP受到编码的限制,即,报文使用7bite的ASCLL码形式,如果存在其他的都,如二进制文件,则需要重新编码之后再发送
HTTP,每个对象封装在一份响应报文中;SMTP所有对象封装在一个报文中

邮件访问协议

POP3
IMAP
基于web的电子邮件

DNS

域名系统:一种主机名到IP地址的转换服务
通常被其他应用层协议所使用

DNS工作概述

1.DNS查询报文通过UDP发送

2.关于集中式DNS服务器设计的不可行性:
单点故障
通信容量
远距离带来的高时延
难维护性
因此必须采用分布式方案

3.三类DNS服务器:
根DNS服务器
顶级域DNS服务器
权威DNS服务器

4.两种查询方式:迭代查询与递归查询

P2P应用
